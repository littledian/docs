# npm包的版本

## 版本格式

主版本号.次版本号.修订号

* 主版本号: 只有在增加了无法向下兼容的API的时候才递增，可以理解为改版；
* 次版本号: 只有在增加了可以向下兼容的API的时候才递增，可以理解为feature版本；
* 修订号: 只有在增加了向下兼容的修正的时候才递增，可以理解为bug fix版本；

## 先行版本

* alpha 内测版本
* beta 公测版本
* rc 正式版本的候选版本

注：可以通过命令:
```bash
npm version prerelease --preid alpha
```
基于当前版本创建一个先行版本，preid参数指定了先行版本的类型

## 版本范围控制

版本的范围通过[semver](https://github.com/npm/node-semver)来控制

* ^ 固定第一个***非零***版本号，后续版本号不小于规定的版本号
* ~ 只会升级最后一位bug fix版本
* version 固定版本号
* latest 最新版本
* <=、>、
